# Why I'm Pursuing Web Development Roles

## The Practical Reality

**Primary driver: Moving to Canada**: My priority is immigrating to Canada. Moving internationally is expensive, and I need stable employment to make it happen. I have 2+ years of production web development experience, which means I can:
- Get hired in senior roles (not entry-level wages)
- Start contributing immediately (no learning curve)
- Save money for future transitions if needed
- Establish myself professionally in Canada

**This is strategic, not cynical**: I could pursue AI/robotics roles directly, but they're harder to get (more competitive, often want advanced degrees), and I'd likely start at lower levels. Web dev roles let me enter Canada with financial stability.

## What I Value in Web Development

### 1. Transferable Skills That Matter Everywhere

**The surface changes, fundamentals don't**:
- Problem-solving and system architecture
- Testing patterns and code quality
- Time management and prioritization  
- Understanding project structure
- Debugging and performance optimization

These skills transfer to AI, robotics, embedded systems, or any technical field. Time spent developing them in web dev context is valuable regardless of where my career goes.

### 2. Leadership Opportunities

**Leadership transcends technology**: Whether I'm leading web developers, AI engineers, or robotics teams, the core skills are identical:
- Bringing out the best in people
- Creating psychological safety
- Clear communication
- Strategic thinking about projects
- Mentoring and teaching

I value senior web dev roles because they offer leadership responsibility. That's something I'll invest significant time developing because it matters in every technical domain.

**My "team dad" leadership style** is equally valuable in web development, AI research, or robotics engineering.

### 3. AI Integration in Web Applications

**Web development is a useful frame for AI work**: My real interest is AI/ML, but web applications are where AI gets deployed to users. WhatNow, moh-ami, Folio - all are AI systems delivered through web interfaces.

**This isn't compromise, it's practical**:
- LLMs need web UIs for users to interact with them
- RAG systems live in web applications
- ML models get deployed through web APIs
- Recommendation systems power web experiences

Web development is how AI reaches people. It's a bridge I'm genuinely interested in crossing well, even if my ultimate passion is the AI itself.

### 4. Things I Genuinely Enjoy About Web Development

**Communication between distant teams**: I excel at bridging technical and non-technical stakeholders. The Integrations Dashboard succeeded because I could understand both backend developers and sales teams. This cross-functional communication skill is deeply satisfying.

**Creating well-designed experiences**: UX is communication between user and computer. My art background makes me care about this. When someone uses an interface I built and it just works intuitively - that's rewarding.

**Immediate visible impact**: Web development has fast feedback loops. You build something, deploy it, users interact with it immediately. This is satisfying in ways that longer research cycles aren't.

## What I'm Committing To

### Full Commitment, Not "Just Passing Through"

**I'll be excellent while I'm here** because:
- The skills I develop transfer to my long-term goals
- Leadership opportunities are genuinely valuable to me
- I care about delivering quality work (my reputation matters)
- Being valuable to my team is how I look after people

**Not mentally checked out**: I won't be collecting a paycheck while dreaming of robotics. I'll be genuinely invested in projects, team success, and delivering value.

### How Long I'll Stay

**Honest answer**: As long as I'm developing meaningful skills.

**What that means**:
- If the role offers leadership growth: I'll invest years developing that
- If there's AI integration work: That's exactly what I want
- If I'm valued and growing: I'm committed
- If it's stagnant with no growth: We should both reassess

**Not a timeline person**: I don't think "exactly 3 years then leave." I think "Am I growing and providing value?" Quality of opportunity matters more than predetermined timelines.
